# Initial Conditions

MOSE initialises the flow field exclusively from **file-based initial conditions**: a structured file containing both the computational mesh and the cell-centred flow variables for all blocks. There is no inline specification of the flow field inside `input.ini`.

IC files are read via the **[ORION](https://github.com/MarcoGrossi92/ORION)** I/O library. ORION supports multiple formats, all of which MOSE can use:

| Format | Extension | Notes |
|--------|-----------|-------|
| Tecplot ASCII | `.tec` | Human-readable; default in bundled test cases |
| Tecplot Binary | `.szplt` | Compact and fast; requires TecIO |
| VTK | `.vts`, `.vtm` | Open standard; suitable for large datasets |

!!! info "IC files and ATLAS"
    In the normal Hydra workflow, the IC file is generated by the **ATLAS** pre-processor. ATLAS takes a mesh, a prescribed flow state, and the thermodynamic data as inputs and writes the IC file in the format MOSE expects. If ATLAS is not available, the IC file can be prepared manually as long as it conforms to the expected format (see below).

---

## File Location and Naming

For single-grid cases the IC file is expected at:

```
INPUT/ic.tec          ← Tecplot ASCII (default)
INPUT/ic.szplt        ← Tecplot binary
INPUT/ic.vts          ← VTK structured
```

The format is inferred from the file extension by ORION.

---

## File Format

Regardless of the format, every IC file stores both the **node coordinates** and the **cell-centred flow variables** for each block in a single file — no separate mesh file is used. The variable layout and ordering described below apply to all supported formats.

### Tecplot ASCII (`.tec`)

The most common format for bundled test cases. Human-readable and easy to inspect or generate manually.

### Tecplot Binary (`.szplt`)

Compact binary format produced by TecIO. Recommended for large multi-block cases where ASCII I/O becomes a bottleneck. Requires ORION to be built with TecIO support (`--use-tecio`). The variable list and zone structure are identical to the ASCII format.

### VTK Structured (`.vts` / `.vtm`)

Open-standard XML-based format readable by ParaView and VisIt. Multi-block cases use a `.vtm` (multi-block) container referencing individual `.vts` (structured grid) pieces. Variable names and layout follow the same conventions as the Tecplot formats.

---

## Required Variables

Every IC file must contain the same set of variables:

| Variable | Description | Units |
|----------|-------------|-------|
| $x,\, y,\, z$ | Node coordinates | m |
| $\rho(i)$ | Partial density of the i-species | kg/m³ |
| $u,\, v,\, w$ | Velocity components | m/s |
| $p$ | Pressure | Pa |

If Spalart-Allmaras turbulence model is used, the IC file must also include:  

| Variable | Description | Units |
|----------|-------------|-------|
| $\tilde{\nu}$ | Spalart-Allmaras working variable | m²/s |

If $k-\omega$ or SST turbulence model is used, the IC file must also include:  

| Variable | Description | Units |
|----------|-------------|-------|
| $k$ | Turbulent kinetic energy | m²/s² |
| $\omega$ | Specific dissipation rate | 1/s |

---

## Restart

When resuming from a previous run (`newrun = false` in `input.ini`), MOSE reads the initial condition from the latest solution file in `OUTPUT/` instead of from `INPUT/`. The solution files share the same format as the IC file (whichever was used) and are loaded through ORION. The flow field, physical time, and iteration counter are all restored.
