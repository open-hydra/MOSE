# Boundary Conditions

Boundary conditions (BCs) in MOSE are specified through an input file that defines the type and parameters of each BC for every block face cell.

!!! info "BC files and ATLAS"
    In the normal Hydra workflow, the BC file is generated by the **ATLAS** pre-processor. ATLAS takes a mesh, a prescribed BC state, and the thermodynamic data as inputs and writes the BC file in the format MOSE expects. If ATLAS is not available, the BC file can be prepared manually as long as it conforms to the expected format (see below).

---

## Multi-grid Naming Convention

When multigrid is enabled, one BC file per grid level is required:

| File | Level |
|------|-------|
| `INPUT/bc_x1.txt` | Finest grid |
| `INPUT/bc_x2.txt` | 2× coarser |
| `INPUT/bc_x4.txt` | 4× coarser |
| `INPUT/bc_x8.txt` | 8× coarser |

For single-grid cases the file is simply `INPUT/bc.txt`.

---

## Face Numbering

For a block with $n_i \times n_j \times n_k$ cells the six faces are labelled:

| Face index | Location | Varying indices |
|:----------:|----------|-----------------|
| 1 | $i = 1$ (imin) | $j = 1 \ldots n_j$, $k = 1 \ldots n_k$ |
| 2 | $i = n_i$ (imax) | $j = 1 \ldots n_j$, $k = 1 \ldots n_k$ |
| 3 | $j = 1$ (jmin) | $i = 1 \ldots n_i$, $k = 1 \ldots n_k$ |
| 4 | $j = n_j$ (jmax) | $i = 1 \ldots n_i$, $k = 1 \ldots n_k$ |
| 5 | $k = 1$ (kmin) | $i = 1 \ldots n_i$, $j = 1 \ldots n_j$ |
| 6 | $k = n_k$ (kmax) | $i = 1 \ldots n_i$, $j = 1 \ldots n_j$ |

For 2-D cases (`nk = 1`), faces 5 and 6 are degenerate. They should be set to the null type (`type = 0`) or symmetry (`300`).

---

## File Format

`INPUT/bc.txt` is the raw, face-by-face boundary condition file read directly by the solver. It is the ground truth from which the solver builds its BC data structures. 

The file is structured as a list of information for every boundary cell. 

The format is as follows:

```
<block>  <i>  <j>  <k>  <face>  <type>
<data line (if required by type)>
```

**Line 1 — face identifier:**

| Field | Description |
|-------|-------------|
| `block` | 1-based block index |
| `i`, `j`, `k` | Cell indices of the boundary cell (1-based) |
| `face` | Face index of that cell (1–6, see [Face Numbering](#face-numbering)) |
| `type` | Numeric BC type code (see table below) |

All boundary cells are listed — one entry per face cell, face by face, block by block.

**Line 2 — BC data (type-dependent):**

Some BC types carry additional data on the second line (see [BC Types](#bc-types)). For types that carry no data (symmetry, extrapolation, axisymmetry), the second line is **absent**.

## BC Types

The following table lists all numeric BC type codes used in `bc.txt`, their physical meaning, and the format of the required data line.

!!! info "BC types and ATLAS"
    More details on these BC types and their parameters can be found in the ATLAS documentation, which defines the expected format and semantics of the BC file. The above table serves as a reference for users preparing BC files manually or interpreting the output from ATLAS.

### Connection

| Type | Name | Data line (comma-separated) |
|:----:|------|-----------------------------|
| `0` | Null / unused | — |
| `101` | Block connection | `bs  is  js  ks  fs  d11  d12  d21  d22` |
| `102` | Chimera overset | `n1  n2` then `n1+n2` donor lines: `bD iD jD kD  vol_frac` |
| `103` | Coupled wall | `bs  is  js  ks  fs  d11  d12  d21  d22` |

`d11, d12, d21, d22` are the $2 \times 2$ index-direction cosine matrix mapping source to target indices.

### Euler Boundaries

| Type | Name | Data line |
|:----:|------|-----------|
| `200` | Axisymmetry | — |
| `300` | Symmetry / slip wall | — |
| `400` | Extrapolation | — |

### Inlet / Outlet

| Type | Name | Data line (comma-separated) |
|:----:|------|-----------------------------|
| `401` | Inlet — stagnation ($T_0$, $p_0$) | `T0,  p0,  alpha,  beta,  rel_fac,  [massf_1 … massf_ns],  [turb_vars]` |
| `402` | Inlet — time-varying $p_0(t)$ | `T0,  p0_file,  alpha,  beta,  rel_fac,  [massf],  [turb]` |
| `403` | Inlet — mass flux + $T_0$ | `T0,  g,  alpha,  beta,  rel_fac,  [massf],  [turb]` |
| `404` | Inlet — mass flux + $T$ static | `T,   g,  alpha,  beta,  rel_fac,  [massf],  [turb]` |
| `405` | Supersonic inlet ($M$, $T$, $p$) | `mach,  T,  p,  alpha,  beta,  rel_fac,  [massf],  [turb]` |
| `406` | Outlet — back pressure | `p_back,  rel_fac` |
| `407` | Ambient ($T_0$, $p_0$, $p_\text{back}$) | `T0,  p0,  p_back,  alpha,  beta,  rel_fac,  [massf],  [turb]` |
| `410` | Assigned state (time-varying file) | `filename  [periodic]` |
| `420` | Choked nozzle | `mach_dummy,  T0,  p0,  p_sub,  p_sup,  r_throat,  rel_fac,  [massf],  [turb]` |

**Common fields:**

| Field | Description |
|-------|-------------|
| `alpha` | In-plane flow angle [rad] or the token `normal` (face-normal inflow) |
| `beta` | Out-of-plane flow angle [rad] or `normal` |
| `rel_fac` | Relaxation factor (0–1; use 1.0 for no relaxation) |
| `massf_s` | Mass fraction of species $s$ (multi-species only; one value per species) |
| `turb_vars` | Turbulence quantities ($k$, $\omega$ or $k$, $\varepsilon$) |

### Viscous Walls

| Type | Name | Data line (comma-separated) |
|:----:|------|-----------------------------|
| `301` | Adiabatic / prescribed heat flux | `q_wall,  k_rough,  eps_wall` |
| `302` | Isothermal wall | `T_wall,  k_rough,  eps_wall` |
| `303` | Isothermal wall + radiation | `T_wall,  q_rad,  k_rough` |
| `304` | Radiative heat flux | `q_rad,  k_rough` |

| Field | Description |
|-------|-------------|
| `q_wall` | Wall heat flux [W/m²] (positive = into fluid; set to `0.0` for adiabatic) |
| `T_wall` | Wall temperature [K] |
| `q_rad` | Radiative heat flux [W/m²] |
| `k_rough` | Sand-grain roughness height $k_s$ [m] (`0.0` for smooth wall) |
| `eps_wall` | Wall emissivity (0–1) |

### Physical Models

| Type | Name | Data line |
|:----:|------|-----------|
| `501` | Manifold connection | `bs  fs` |
| `502` | SRM grain boundary | `T_af,  a,  n,  p_ref,  rho_grain,  SF_geo,  [massf_1 … massf_ns]` |

!!! tip
    More details on these BC may be found in the [Theory Guide](../theory/index.md)

## Examples

**1-D Case (100 cells, all faces extrapolation)**

```
       1       1       1       1       1     400
       1     100       1       1       2     400
       1       1       1       1       3     400
       1       2       1       1       3     400
...
       1     100       1       1       4     400
```

Faces 1 and 2 each have 1 entry (the single cell at the end of the 1-D domain). Faces 3 and 4 each have 100 entries (one per $i$-cell).

**2-D Supersonic Inlet (type 405)**

```
       1       1       1       1       1     405
    0.300000E+01,    0.100000E+01,    0.100000E+01,         normal,         normal,    0.100000E+01,    0.100000E+01,
       1       1       2       1       1     405
    0.300000E+01,    0.100000E+01,    0.100000E+01,         normal,         normal,    0.100000E+01,    0.100000E+01,
```

**Block Connection (type 101)**

```
       1       1      16       1       4     101
       2       1       1       1       3       1       0       0       1
```

The data line carries the target block and face information plus the direction cosines.
